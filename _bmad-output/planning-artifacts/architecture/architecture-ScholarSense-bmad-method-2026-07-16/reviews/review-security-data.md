# 安全、合规与数据完整性联合评审

## 修复后复审（2026-07-16）

**Verdict：PASS。残余 Critical：0；残余 High：0。**

本轮针对上一版五组问题逐项复审，架构不变量已形成闭环：

1. **导出撤权：已关闭。** AD-13 明确生成前和每次下载按当前授权重检，旧权限快照只能与当前权限取交集；AD-24 又禁止持久化明文授权结果并要求读取/下载时应用当前策略；AD-25 要求更正后立即撤销已生成文件。结合 AD-8 的逐请求对象授权与 AD-9 的字段三态投影，旧授权不能继续放宽下载结果。
2. **主体/事实更正级联：已关闭。** AD-4 固定永不复用的 `StudentRef`、版本化映射和不可改写历史；AD-25 以 `supersedesId/reason/effectiveAt/lineageId` 驱动事实、评估、候选、线索、读模型、缓存和导出的级联失效/重建，并以消费者 watermark 与对账作为完成判据。该规则足以防止各模块局部合规却跨学生串档或重复生成。
3. **审计读路径与防篡改：已关闭。** AD-10 要求敏感查看、检索、下载在返回内容前持久化本地审计事实，失败不返回；关键命令与本地审计同事务。集中 ledger 使用独立 writer、禁止 update/delete、连续 sequence 与 previousHash，并定义归档失败重放、积压阈值及高风险默认拒绝，覆盖了原先读路径无原子边界和选择性漏归档问题。
4. **CryptoPort：阻断问题已关闭。** AD-9 固定解密边界，禁止领域事件、outbox、日志、读模型和缓存携带非必要明文；端口调用包含服务身份、数据分类、用途、keyRef、traceId，密文携带 keyVersion，轮换采用双读新写，密钥故障失败关闭且不得降级明文。算法与产品继续 Deferred 不会让下游形成相反的安全语义。
5. **恢复一致性集：已关闭。** AD-14 将数据库、outbox、对象元数据、审计序列、密钥引用/版本与外部 ID 映射绑定为同一恢复一致性集，恢复后必须完成序列、水位、对象和解密性对账再开放写流量；RPO/RTO 由半年完整演练和业务可处理时间验收，不再只是组件启动目标。

### 残余非阻断风险

- **Medium — 导出访问令牌的具体失效机制尚属实现契约。** 当前不变量已要求每次下载实时授权，因而安全语义完整；实施时仍应验收对象存储不向客户端签发可绕过应用授权的长效直链，并验证撤权后所有未消费令牌失效。这是测试/适配器契约，不需新增平台级 AD。
- **Medium — CryptoPort 的密文替换防护需在密码方案中落地。** 当前 `purpose + data classification + keyRef + service identity` 已固定跨模块语义；校方方案确认时应验证认证上下文包含环境、聚合/对象和字段身份，防止密文跨用途替换，并覆盖备份、临时对象与导出制品加密。
- **Medium — 更正风暴与消费者永久失败需运行手册。** AD-25 已以 watermark 和对账定义正确性完成条件；实施需规定消费者长期无法追平时的隔离、告警、人工处置及导出/治理动作保持禁用的策略。

### 复审要求的最小验收证据

1. 撤销角色、责任学生或协同任务后，旧下载链接与在途导出均被拒绝；字段权限降低后不能取得旧明文。
2. 主体合并、拆分、改绑和源事实更正的端到端测试，证明所有消费者 watermark 收敛、当前主体视图唯一且历史审计链不被改写。
3. 敏感读审计持久化失败、集中 ledger 中断及归档中断的故障注入，证明返回阻断、事实可重放和 sequence/previousHash 缺口可检测。
4. 密钥轮换与密钥服务不可用测试，证明双读新写、旧密文可解和失败关闭。
5. 从同一恢复点恢复数据库、outbox、对象、审计、keyVersion 与外部映射，完成对账后才开放写流量，并在 RPO≤15 分钟、RTO≤2 小时内恢复核心任务处理。

## 结论

**Verdict：有条件不通过（需修复 2 个 High 后方可定稿）。**

脊柱已经建立了较强的局部不变量：服务端组合授权、字段三态投影、事务内 audit/outbox、不可变外部事实、质量门、人工决策以及 RPO/RTO 目标。但两个跨不变量的组合缝隙仍允许“各模块都按各自规则实现，却产生越权或事实分裂”：导出制品的持续授权，以及主体映射/源事实更正的级联一致性。

## High

### H1 — 导出权限快照可绕过即时撤权，已生成文件缺少持续授权与强制失效语义

- **涉及：** AD-8、AD-9、AD-11、AD-13。
- **漏洞：** AD-8 要求每次请求重算授权；AD-13 却只要求导出作业保存“字段权限快照、文件有效期和下载审计”。合规实现可以在申请时按权限生成含敏感明文的文件，随后用户责任学生范围、角色或协同任务被撤销，但下载端只验证作业所有者和未过期 URL，继续交付旧制品。对象存储预签名 URL、消息中的下载链接或已进入缓存/CDN 的文件还可能绕过应用层的逐请求授权。此时 AD-8、AD-9 和 AD-13 各自均可声称满足，组合结果仍是撤权后数据泄露。
- **影响：** 即时撤权失效；临时协同授权可能转化为持续离线数据占有；字段级权限降低后仍可取得旧明文；审计只能事后证明泄露，不能阻断。
- **建议处置：autofix。** 在脊柱绑定导出的“双时点授权”与制品失效：申请/生成时固定最小字段投影，**每次下载时**重新校验当前主体、对象范围、字段权限、审批与任务授权；当前权限只能缩小结果，不能由旧快照放宽。撤权、审批撤销、对象范围变化或字段敏感等级上调必须立即使未下载制品和所有访问令牌失效。对象存储不得向客户端暴露可脱离授权网关长期使用的地址；短时一次性令牌须绑定用户、作业、制品版本和用途。若无法安全裁剪既有制品，则销毁并按当前权限重生成。下载审计与令牌消费需具备原子或可证明的幂等关系。

### H2 — 主体映射与源事实更正没有统一级联失效协议，可形成跨学生串档和旧线索继续有效

- **涉及：** AD-4、AD-5、AD-6、AD-11、AD-19，以及 `subject-registry`、`ingestion-quality`、`signal-evaluation`、`clue-care`、`reporting` 的边界。
- **漏洞：** AD-4 允许追加“更正事件”，AD-5 将不可唯一主体隔离，但没有规定主体映射被拆分、合并、改绑或权威源回补后，既有 `NORMALIZED_FACT → RULE_EVALUATION → CANDIDATE/CLUE → EVIDENCE_SNAPSHOT → READ MODEL/EXPORT` 如何传播失效、重归属和重算。一个完全合规的实现可保留原记录不可变、追加新映射，同时旧候选/线索仍指向错误 `STUDENT_REF`，新事实又在正确学生下生成第二份记录；仪表盘、档案、协同转介和导出于是各自读取不同版本。AD-6 的去重键又以“学生”为组成部分，映射改绑后无法阻止跨旧/新主体重复候选。AD-18 的人工确认也不能修复证据归属错误，反而可能让人工决定建立在错误主体上。
- **影响：** 跨学生隐私泄露、错人关怀或转介、重复线索、历史统计双计/漏计、删除或更正无法贯穿派生数据，且审计难以回答“当时使用的是哪个主体映射版本”。
- **建议处置：autofix。** 增加版本化数据血缘与更正协议：所有规范事实、评估、候选、线索证据、标签、读模型行和导出必须携带稳定源记录 ID、规范事实版本、主体映射版本与派生链 ID；映射/源更正发布单调版本的 correction/remap 事件。各拥有模块必须幂等地把受影响派生物标为 `superseded/identity-disputed/evidence-invalid`（与业务状态正交），阻断新动作与导出，重建读模型并按已定义策略重算。禁止静默移动已人工处置的正式线索：应保留原审计链，以受控纠错命令完成撤销关联、重新归属或生成替代线索，并通知当前责任人/协同方。重算完成前不得清除隔离；需有端到端对账证明同一派生链只落在一个当前主体上。

## Medium

### M1 — “append-only + 归档防篡改”不足以证明在线审计未被删改或选择性漏归档

- **涉及：** AD-10、AD-7。
- **问题：** 审计与业务事务同提交只覆盖共享数据库内的业务命令；登录、拒绝、查看、检索等读路径没有业务事务可绑定。`append-only` 若仅靠应用权限实现，数据库管理员或被攻陷的高权限账户仍可更新/删除；归档前窗口及“哪些记录成功归档”的完整性也没有证明机制。外部防篡改存储失败时，正文没有规定阻断高风险动作、持久补偿还是告警升级。
- **建议处置：autofix。** 区分写命令审计与读/拒绝审计的提交语义；绑定数据库级禁止 UPDATE/DELETE、独立写入身份、单调序列/哈希链或批次 Merkle 摘要、校方可信时间戳/签名、归档回执和连续性对账。高风险动作在审计不可持久化时应 fail closed；普通读取至少进入本地持久队列并告警，明确最大未归档窗口。审计事件 schema 还应记录策略版本、字段投影/导出范围、主体映射与数据版本，且避免 IP 等个人信息超期保留。

### M2 — CryptoPort 只统一调用入口，未约束明文边界、密钥上下文和轮换失败行为

- **涉及：** AD-9、Deferred“密码与密钥实现”。
- **问题：** 两个模块都可“经 CryptoPort”加密，但独立选择字段粒度、用途上下文、AAD、确定性模式、密钥版本和轮换策略，导致密文可跨租域/字段替换、搜索索引泄露、备份或临时文件留明文。Deferred 可以推迟算法和产品，却不应推迟这些互操作和安全边界。
- **建议处置：defer implementation, autofix invariant。** 脊柱应先绑定：明文只在获授权的应用用例最短生命周期内出现；密文绑定环境、模块、聚合、字段、主体和版本的认证上下文；密文携带算法/密钥版本；轮换可双读单写且可恢复；密钥不可导出到业务进程或日志；导出、对象存储、备份、队列和临时文件同样受保护；CryptoPort/KMS 不可用时敏感写入与导出 fail closed。具体 SM/AES/KMS 产品仍可 Deferred。

### M3 — 规则、标签与质量状态虽被定义，缺少治理状态改变对在途计算和已生成结果的统一生效点

- **涉及：** AD-5、AD-6、AD-13、AD-19。
- **问题：** 长任务保存输入快照，但规则熔断、版本回退、标签争议暂停或质量状态在任务执行期间发生变化时，未绑定提交候选前必须复验哪个版本/状态。合规 worker 可以用启动时的“生产/合格”快照，在规则已熔断或质量已降级后仍提交候选；相反，历史快照又要求可复现，两者缺少明确的提交栅栏。
- **建议处置：autofix。** 计算可按不可变快照复现，但任何候选/线索创建提交必须复验当前规则发布纪元、质量门版本、主体映射版本及熔断状态；纪元变化即丢弃或转入隔离结果，不得正式发布。标签争议/撤销应作为正交治理状态传播到投影和授权，不静默改写历史。

### M4 — RPO/RTO 是目标而非可验证恢复契约，且未覆盖对象存储、密钥、审计和外部映射的一致恢复点

- **涉及：** AD-14、AD-7、AD-10、Deferred“灾备物理方案”。
- **问题：** PostgreSQL 恢复到 T-15 分钟，而对象存储、密钥版本、审计归档和外部工单双方 ID 映射若恢复到不同时间点，可能出现数据库引用不存在的证据/导出、恢复后重复发送外部工单、或密文无法解密。满足单个组件 RPO/RTO 不等于业务闭环可恢复。
- **建议处置：defer physical design, autofix acceptance contract。** 定义业务一致恢复点及恢复顺序；对象、数据库、密钥元数据、审计锚点、outbox/consumer 去重账本纳入同一恢复清单。要求定期演练、恢复后血缘/引用完整性检查、outbox 重放和外部对账；RTO 结束条件应是关键业务用例与审计可用，而非数据库启动。明确备份加密、不可变性和恢复授权。

## Low

### L1 — “线索而非结论”未明确禁止模型输出进入授权、排序容量和质量门等间接高影响控制面

AD-18 禁止自动改变等级、转介和重大动作，但智能输出若进入工作台优先排序、督办资源分配、规则发布建议自动采纳或质量门参数，仍可能间接决定谁被看见、谁被延迟。建议将模型/生成式输出限定为带来源和置信说明的非权威建议，禁止成为授权、字段投影、主体映射、质量门、熔断恢复和 SLA 优先级的唯一输入；所有采用动作记录模型/提示/知识版本与人工决定。

## 通过后应具备的最小证据

1. 撤权后旧导出链接、对象存储令牌及未下载制品全部不可用的集成测试。
2. 主体 A→B 改绑与一条源事实更正的端到端测试，证明旧派生物失效、当前视图唯一、正式线索纠错保留完整审计。
3. 审计写入/归档故障注入，证明高风险动作阻断且审计序列无缺口。
4. 规则熔断与质量降级恰发生在 worker 提交前的竞态测试。
5. 全栈时间点恢复演练，证明数据库、对象、密钥、审计、outbox 和外部工单映射一致。

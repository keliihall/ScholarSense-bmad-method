# UX 架构约束提取

> 来源：最终版 `DESIGN.md`、`EXPERIENCE.md` 及 UX `.memlog.md`。本文只提取已明确的架构约束，不提出实现设计。若原型或导入物与 UX 双脊柱冲突，以双脊柱为准。

## 1. 前端形态与宿主边界

- 产品由两种宿主形态组成：学校统一门户内的响应式 Web 管理端，以及学校统一 App WebView 内的移动轻量处置。
- 前端继承 Vue 3、Element Plus、ECharts，并复用学校 SSO、公共消息、公共待办、门户主题与移动壳层。
- 移动端与 Web 共用唯一业务状态；移动端不创建独立品牌壳、独立底部导航，也不承载领导驾驶舱、规则配置或复杂数据质量页面。
- Web 需覆盖 1440×900 基准桌面与 1366×768 最低桌面；移动以 375px 为基准且不得产生横向页面滚动。
- `[ASSUMPTION]` 产品暂按桌面、窄桌面/平板、移动三档响应行为；768px、1024px 阈值尚未由学校规范确认。
- 浏览器验收范围为 Edge、Chrome 最新两个稳定大版本，以及学校统一 App 当前生产支持的 iOS/Android WebView。
- 页面/视图必须承载可恢复的筛选、排序、分页、对象与来源上下文；大数据列表使用分页，不使用无限滚动。
- 图表与等价表格/数值摘要共享筛选和指标口径，切换不得丢失状态。

## 2. 服务端必须支持的状态模型

- 所有正式事项必须具有责任人、时限、业务状态、结果及可追踪状态变化；成功反馈必须返回新状态和下一步，不能只返回瞬时成功消息。
- 候选初审与首次核实的时钟均从候选生成时开始，接纳、责任转移不得重置；完成后仍需保留“曾超期”。时限分别为：III/II/I 级初审 2 小时、8 小时、2 个工作日；首次核实 24 小时、48 小时、5 个工作日，并能返回绝对截止时间、剩余时间与适用工作日口径。
- 转介工单状态至少覆盖：待接收→处理中→已回填→已关闭，并包含待补充、退回；每次变化记录操作者、时间、说明与下一责任人。工单关闭不自动关闭原线索。
- 责任转移必须保留原截止时间，记录原/新责任人、原因、生效时间及未完成事项；双方收到同一事项更新，失败进入可追踪异常队列。
- 核实反馈根据“属实/噪声/待观察”驱动不同必填项；噪声不得写入属实历史。机器信号、人工核实、协同结果必须保持类型可辨。
- 数据质量不达标时必须支持熔断：熔断期不生成正式线索；既有线索只更新证据质量。恢复需经过完整性校验、样本重算与授权确认，进入恢复观察；不得补发已失去时效的线索，也不得改写既有线索历史。
- 规则、标签、模型/策略必须支持不可覆盖的历史版本。生产规则通过新版本发布；标签支持待审核、生效、过期、撤销、争议暂停；模型/策略发布、扩大灰度、回退需形成持久审批与审计轨迹。
- 数据表面需返回统计周期、口径、更新时间、来源及必要的数据质量快照；陈旧或依赖不可用时需标识受影响数据和动作，不能把旧数据伪装成实时数据。
- 工作纪实只汇总已记录工作动作、周期和异常说明，不得进入学生线索/风险计算，也不得形成个人绩效排名。

## 3. 权限、隐私与审计约束

- 菜单、首页、路由、对象、字段和导出均必须依据常设角色与对象级授权执行；仅隐藏菜单不构成权限控制。
- 敏感字段存在明文、脱敏、不可见三态；权限变化在下次访问立即生效。不可见响应不得泄露对象是否存在、字段名、字段值或值长度；不可见字段不进入无障碍树。
- 对象授权失效后须立即撤销访问；越权访问需要记录审计，并以不泄露对象信息的方式响应。
- 校级领导驾驶舱默认只提供汇总态势，不提供临时个体下钻；自然语言问数只查询口径库中的已定义汇总指标，敏感或越权问题必须拒答且不泄露对象存在性。
- 转介仅暴露完成任务所需的最小必要摘要；临时授权过期时以服务端最新权限与状态为准。
- 审计至少支持按用户、对象、动作、时间、结果、traceId 检索；原始审计记录只读且至少保留 6 个月。
- 导出必须继承当前字段权限并最小化范围；汇总数据陈旧或依赖不可用时，可禁用基于失真数据的导出与治理行动。
- 高风险动作包括规则发布/回退、熔断恢复、权限变更等，必须支持影响预览、理由及相应门禁；哪些动作使用即时确认、二次认证、审批或双人复核仍是 UX 明列的开放问题。
- 多角色权限包是否可叠加、是否允许工作视角切换及其范围确定规则仍是开放问题，不应从原型的演示角色切换推导正式权限模型。

## 4. 同步、并发、幂等与连续性

- Web 与移动端共享唯一服务端业务状态，完成状态跨端同步 P95 不超过 5 秒。
- 并发冲突以服务端状态为准；冲突响应/刷新后需呈现最新操作者和操作时间，同时允许客户端保留未提交草稿，但禁止重复回填或重复完成。
- 核实、转介、治理动作及接口回调必须幂等；提交中应能锁定重复触发，失败不清空表单/草稿，并允许显式幂等重试。
- 移动端网络波动或离线时，可读取带时间戳的缓存摘要；任何状态变更不得在未获服务端确认时宣告成功，草稿需保留并在恢复后显式重试。
- 异步导出、重算、发布不得绑定页面会话；用户离开后仍可从公共待办或任务中心查看任务状态与结果。
- 工单发送失败需进入“待发送/可重试”等真实状态，不得宣告接收方已接收；责任转移失败、无有效接收人等情况需进入可追踪异常队列。

## 5. 深链、会话与导航状态

- 公共待办、统一门户和统一 App 必须能够深链到唯一任务、线索或工单。
- 深链需保持对象、来源和筛选上下文；返回时不得丢失工作台位置。
- 会话失效后重新认证必须返回原目标；重认证过程仍需重新执行当前权限校验。
- 筛选提交后应能由 URL 或等价视图状态恢复；页面需持续显示已选条件与清除入口。

## 6. 导出与异步任务契约

- 大范围导出采用异步生成，状态必须包含任务号、申请范围、审批/失败状态、文件有效期和下载审计。
- 异步导出、重算、发布的失败和重试必须保留筛选/输入，说明影响范围，并提供持久结果入口。
- 不得以 Toast 或单次 HTTP 成功代替导出、发布、督办、转介等业务状态闭环。

## 7. 无障碍对架构/接口的要求

- Web 与 WebView 的产品底线为 WCAG 2.2 AA；状态、等级、错误、必填、临期/超期不得只靠颜色表达。
- 服务端/前端数据模型应能为状态提供文字、语义图标/形状所需的明确语义，并为敏感字段提供明文/脱敏/不可见状态而非仅返回模糊后的字符串。
- 表格需有可访问名称、列头关系、排序状态和键盘行操作；图表必须存在同口径数值摘要或数据表所需的数据。
- 动态提交、问数回答、同步冲突需要可供 live region 宣告的明确结果信息。
- 错误响应需支持字段内错误与顶部错误摘要关联，并能定位首个错误字段。
- 页面切换后焦点进入主标题；浮层关闭后焦点返回触发点；动作不可只在 hover 出现，拖拽不可作为唯一排序方式。
- 尊重 `prefers-reduced-motion`；加载/上传进度需有静态替代且不得闪烁。200% 缩放不得丢失关键内容。
- `[ASSUMPTION]` 最小交互目标为桌面 40×40 CSS px、移动 44×44 CSS px，待学校移动规范确认。

## 8. 性能与时效要求

- 工作台、线索列表和详情在正常负载下，P95 需在 2 秒内呈现可操作内容。
- 汇总看板常用筛选 P95 需在 3 秒内更新。
- Web 与移动业务完成状态同步 P95 不超过 5 秒。
- 超过加载阈值时需支持页面骨架或局部进度，不得冻结整个页面，也不得将陈旧数据伪装为实时结果。
- 单条核实反馈以 3 分钟内完成为体验目标；已有学生、证据、责任信息需可自动带入，只补充人工核实与行动结果。

## 9. 不可继承的原型行为

- 不继承演示角色切换或模拟身份切换。
- 不继承“只隐藏菜单、不拦截路由”的越权实现。
- 不继承固定时间、无分页、无限滚动等演示数据行为。
- 不继承单击循环权限或以客户端展示状态代替真实授权变更。
- 不继承 Toast 假闭环；提交必须进入真实持久状态。
- 不继承自然语言问数未知意图时默认成功或返回看似有效答案；必须区分无法理解、零、缺失、敏感和越权。
- 不继承原型散落的蓝绿色作为品牌主色、11px 核心信息字号、无语义点击块。
- 不继承学生风险排行榜、学院单指标排名、红色告警墙、告警音、发光地图等“预警大屏”模式。
- 不把关怀等级、标签、规则分数或单次成绩/排名/消费变化呈现为学生结论；跨类别证据不得简单相加为更高风险。
- 不在移动端复制桌面驾驶舱、配置页、独立品牌壳或独立底部导航。
- Mockup 仅说明构图和视觉层级；协同转介、学院治理、规则/标签、数据质量、审计/合规、问数、报表、一人一档和工作纪实均为 spine-only，不得从缺少 mockup 推导功能缺失。

## 10. 来源中的待决项（不可当作已定约束）

- 校徽正式矢量、最小尺寸、反白版、安全区尚待校方确认。
- Web 正式圆角、控件高度、栅格与中间断点尚待校方确认。
- 移动规范中 `#FEAE4A` 与 `#478FE4` 的官方语义名称尚待确认。
- 高风险动作的认证/复核级别、多角色叠加与视角切换规则、响应阈值、I/II/III 对外措辞尚未定案。

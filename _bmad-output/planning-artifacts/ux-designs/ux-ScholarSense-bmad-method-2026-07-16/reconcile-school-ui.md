# 学校 UI 输入对账报告

## 对账范围

本报告对照以下输入与当前 UX 双脊柱，记录学校 UI 输入如何被继承、规范化、遗漏、冲突处理或有意舍弃：

- `docs/input/UX/校园服务总入口WEB端设计规范.pdf`
- `docs/input/UX/苏大UI规范说明(1)(1).pdf`
- `docs/input/UX/苏大logo.png`
- `.working/extract-school-ui.md`
- `.memlog.md`
- `DESIGN.md`
- `EXPERIENCE.md`

证据限制：Web PDF 是 `1600 × 41737 pt` 的单张超长画布，只完成低分辨率整览与全文文字提取；精确规则以可提取文字为准。移动 PDF 共 18 页，已对覆盖封面、前言、颜色、字体、图标、控件、留白和底部导航的代表页作视觉核验。校徽 PNG 已直接视觉检查。

## 结论摘要

学校输入的核心视觉身份已进入 `DESIGN.md`：Web/移动品牌红、文字与状态色、字体方向、1440×900 与 375px 基准、移动两档 gutter、校徽使用约束、学校组件家族及统一门户气质均被继承。`EXPERIENCE.md` 则将其转化为响应式双表面、组件行为、状态、键盘/触屏和无障碍契约。

当前双脊柱没有把学校 PDF 当成完整设计系统照搬，而是执行了 `.memlog.md` 的优先级：学校硬规范优先；学校未给数值的圆角、栅格、断点、Logo 安全区明确标为产品假设；与产品形态冲突的独立移动品牌壳和底部导航不继承。

主要未闭合项是：学校 Logo 正式 VI 参数、Web 控件高度/圆角/栅格、移动辅助色的官方语义名。另有若干学校尺寸和完整字号未进入 token；这不阻塞当前脊柱，但若要形成可直接编码的全量设计系统，应补充或说明有意由 Element Plus 继承。

## 二、已规范化或扩展

这些内容不是逐字照抄，而是将学校素材整理成可实现的产品契约。

1. **颜色语义化。** 学校的色板被命名为 `brand-*`、`text-*`、`surface-*`、`border-*`、`status-*`，避免组件内部散落十六进制值。
2. **学校主红与平台分离。** Web `#AF251B` 与移动 `#D03D37` 保留为不同 token，符合 `.memlog.md` 决策，没有错误合并成一个“苏大红”。
3. **字体栈工程化。** 学校只给苹方/SF Pro/系统字体方向；脊柱补入思源黑体和通用 sans-serif 回退。字体粗细将学校 Light/Regular/Middle/Bold 归一为 CSS 数值 `400/500/600` 等实现表达。
4. **间距归一。** 学校示例散见 `6/8/10/12/13/20/24/26/45px`；脊柱采用 4px 基础比例，并仅将有明确通用语义的移动 `10/24px` gutter 单独保留。该 4px 比例是产品规范化，不是校方明确 spacing scale。
5. **组件从外观样例升级为行为契约。** 学校按钮、表单、表格、提示和空状态主要展示视觉状态；`EXPERIENCE.md` 补充键盘、幂等提交、错误聚焦、URL 状态恢复、服务端冲突、权限与深链行为。
6. **无障碍产品化。** WCAG 2.2 AA、非颜色唯一编码、图表等价表格、焦点、读屏、减少动态效果和 200% 缩放来自 PRD/产品补充，而非学校 PDF；已在脊柱中明确作为更高验收底线。
7. **响应式行为产品化。** 学校只明确 1440×900 与 375px；双脊柱新增 1366×768 最低桌面、平板/移动行为以及桌面侧栏和移动卡片策略，中间断点均标注 `[ASSUMPTION]`。
8. **门户导航产品化。** 学校示例中的“首页/一件事/全部事务/全部应用”等通用门户条目没有直接复制为 ScholarSense IA；产品改用学校统一入口 + ScholarSense 角色化表面，并复用 SSO、消息、待办。
9. **校徽安全区谨慎补充。** “直径 1/4”只作为暂定 `[ASSUMPTION]`，同时明确不是校方已确认安全区。
10. **学校状态样式与关怀语义解耦。** 学校红、绿、橙、蓝仅作为辅助状态色；脊柱要求关怀等级写成“核实优先级”，并用文字/图标与时限共同表达，避免把红色变成学生风险结论。

## 五、推断与假设审计

### 已正确标注为 `[ASSUMPTION]`

- `rounded.sm/md/lg = 2/4/8px` 与 `full = 9999px`。
- 校徽透明边界外暂留直径 `1/4` 空白。
- 768px/1024px 的平板与移动行为阈值。
- 移动最小交互目标 `44×44 CSS px`；学校 PDF 只展示部分 `40×40px` 控件，未声明统一触控验收值。
- 学校未提供 Web 栅格、最大内容宽度、完整控件高度与中间断点。
- `#FEAE4A/#478FE4` 色名错位时以色值为准。

### 属于产品补充但当前未在每处重复标注

- 4px spacing scale。
- 1366×768 最低桌面验收。
- WCAG 2.2 AA、40×40 桌面目标、200% 缩放、减少动态效果。
- Element Plus 默认低强度对话框阴影。
- PingFang/SF Pro/Source Han Sans 的合并字体栈及 CSS 字重映射。

这些补充均与学校输入不冲突，但在交付说明中应继续称为 ScholarSense 产品级规范，而非“苏大 UI 硬规范”。

## 六、被丢弃或不继承的定性想法

1. **独立移动品牌壳与底部导航。** 移动 PDF 展示完整“云中苏大”底部导航及两套文案；ScholarSense 仅作为统一 App WebView 轻量处置入口，因此不再造壳、不重复底部导航。
2. **移动首页金刚位与应用服务编排。** 学校示例中的四列应用入口、AI/校园码/一卡通等属于统一 App 首页，不是 ScholarSense 任务表面，故只保留图标系统参考，不进入 IA。
3. **学校示例导航文案。** “首页/一件事/全部事务/全部应用”和校园服务分类不直接进入 ScholarSense 菜单；仅继承顶部导航、面包屑和侧栏模式。
4. **示例性状态插图的具体画风。** 404、暂无网络、暂无日程等状态类型被继承，但示例插图没有作为品牌资产直接推广；脊柱只要求准确标题、说明和唯一下一步。
5. **大面积红色强调的演示气质。** 品牌红被保留，但“红色告警墙、风险排行榜、发光/渐变战情室”等定性方向被明确拒绝。
6. **所有学校字号与尺寸的机械复刻。** 当前双脊柱有意保留产品真正使用的层级，并继承 Element Plus 基础系统；未将 PDF 每个示例尺寸都扩展成 token。
7. **Light 字重作为核心正文。** 学校列出 Light，但高密度、高敏感工作流更重视可读性；当前正文使用 400，标题/强调使用 500/600。
8. **仅用 Toast 或视觉状态表示完成。** 学校输入没有提供完整持久化闭环；产品明确要求成功反馈说明新状态和下一步，Toast 不替代真实状态。
9. **校徽作为页面装饰反复出现。** 校徽只用于统一门户品牌识别，不在模块页面重复堆叠。

# 学林知微平台规范契约

## Why

学林知微要把教务、学工、后勤、保卫等系统中的细微变化转化为可解释、可核实、可跟进的关怀线索，让辅导员更早发现值得关心的变化，同时保护学生尊严、避免把数据变化当作结论，并让学院、学工部门和学校领导能够基于可信汇总持续治理。

## Capabilities

- **CAP-1**
  - **intent:** 用户通过统一门户进入与其职责匹配的授权工作面，并获得正确的责任学生、菜单和公共待办。
  - **success:** 权威身份和责任关系同步后，授权范围、深链和越权审计结果一致；未授权访问不泄露对象存在性。
- **CAP-2**
  - **intent:** 系统汇聚校内信号并生成带证据、个人基线、场景排除和数据质量信息的候选关怀线索。
  - **success:** 正式线索缺少来源、时间窗、当前值、基线或质量快照时无法进入工作队列；质量门槛不满足时相关规则熔断。
- **CAP-3**
  - **intent:** 辅导员能够按时限优先处理候选初审、人工核实、关怀行动和持续观察。
  - **success:** 候选接纳、责任转移均不重置原时钟；每条正式线索都有责任人、时限、状态、结果和可复盘时间线。
- **CAP-4**
  - **intent:** 责任人员能够从原线索发起校内协同转介，并由协同部门处理和回填结果。
  - **success:** 转介状态为可追踪的接收、处理、回填、关闭或退回补充；回填回到原线索，关闭转介不自动关闭原线索。
- **CAP-5**
  - **intent:** 学院、学工部门和领导能够查看授权范围内的运行态势、超期积压、噪声和资源压力并采取督办行动。
  - **success:** 每个指标显示口径、周期和更新时间；可下钻的汇总能回到治理动作，系统不生成学生风险排行榜或学院单指标排名。
- **CAP-6**
  - **intent:** 数据责任人和治理人员能够管理数据质量、规则/标签/策略版本、熔断恢复和运行异常。
  - **success:** 发布、回退、恢复和权限变更均经过影响预览、复核门禁、幂等持久化和审计；修复后不补发已失去时效的线索。
- **CAP-7**
  - **intent:** 用户能够在响应式 Web 与移动轻量处置入口完成适合设备的任务，并保持同一业务状态。
  - **success:** Web/WebView 状态同步 P95≤5 秒；移动端支持待办、证据摘要、联系、快速核实、观察和协同进度，不承载驾驶舱或复杂治理页。
- **CAP-8**
  - **intent:** 授权人员能够检索审计、处理合规材料、按权限异步导出和查询受控汇总指标。
  - **success:** 字段级可见性在读取和下载时强制执行；导出有范围、审批/失败状态、有效期和下载审计；敏感或越权问数明确拒答。

## Constraints

- 线索而非结论：所有规则或模型输出必须经过人工核实，文案不得诊断或给学生贴负面标签。
- 采用模块化单体 + 六边形端口适配器；领域模块只能通过端口和事件协作，禁止跨模块表写入。
- Candidate 与 Clue 在 clue-care 聚合内原子维护；规则计算只发布不可变评估结果，不能直接写正式线索。
- 本地审计事实与业务事务同库原子写入并经 outbox 传递；集中审计账本只追加、可检索，敏感值不得进入日志。
- 正式状态变更必须服务端幂等、乐观并发控制、审计和时钟持续计算；高风险操作默认拒绝，需显式复核。
- 数据质量、权限、用途隔离、最小必要字段和字段级脱敏适用于列表、详情、导出、问数及后台任务。
- 首期核心闭环不依赖生成式模型；智能增强必须可灰度、可评估、可回退。
- Web 按学校统一门户与 UI 双脊柱实现；桌面基准 1440×900、最低验收 1366×768，移动基准 375px，WCAG 2.2 AA。
- 体验目标：工作台/详情 P95≤2 秒可操作内容，汇总常用筛选 P95≤3 秒，单条核实目标≤3 分钟。

## Non-goals

- MVP 不提供学生、家长的直接登录、知情查询、反馈或申诉端。
- 不建设独立身份认证、统一门户、公共消息/待办或源业务系统；仅通过适配器接入。
- 不以规则分数直接形成学生结论、风险排行榜、学院排名或辅导员绩效排名。
- 不在移动端复制领导驾驶舱、规则配置、复杂质量治理和大范围报表能力。

## Success signal

在真实试点中，辅导员可从统一门户进入工作台，在一次任务流内查看证据、完成核实、发起必要协同并看到结果回填；学院与校级管理者能从可信汇总定位超期或质量问题并采取行动。质量异常会阻止错误线索继续产生，所有关键访问和状态变化可审计复盘。

## Assumptions

- 学校统一身份、组织关系、门户、消息待办、数据交换和基础设施按 PRD 所述可供接入。
- 生产前端只绑定技术家族；具体生产版本在验证后选定，原型 lockfile 版本不构成生产基线。

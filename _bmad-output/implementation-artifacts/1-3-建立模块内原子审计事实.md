# Story 1.3：建立模块内原子审计事实

Status: done

> 证据边界：G-01 已批准，架构已冻结 `LocalAuditFact` 的必备语义、UTC/可信时间与模块本地事务边界，因此可以开始实施。PP-1.0.0 已冻结 `server-synchronized-ntp` 与最大偏差 100ms，但仓库尚无独立版本化审计 schema，identity runtime profile 也仍缺 `clockSource` 运行绑定；本 Story 必须将 AD-10/AD-12/AD-16 落为可执行的 v1 契约，并取得运行证据，不得把“契约已冻结”写成“NTP 已验证”。本 Story 只交付本地事实模式、identity-access 参考接入与失败关闭；集中 `AuditLedger`、hash/sequence、归档检索及销毁证明仍分别属于 Story 1.4、1.5 和 6.6。

## Story

作为合规负责人，
我希望让关键命令和敏感读取均可追责，
以便业务结果与本地审计证据不会分裂，审计不可用时也不会泄露或伪造成功结果。

## 业务与交付价值

- 为 FR-8 提供第一个可运行 phase：业务命令、拒绝与敏感读取在各自模块内生成统一、最小、不可原位改写的审计事实。
- 为 NFR-15 提供“原子提交”证据；hash/sequence 完整性和归档证据由 1.4/1.5 继续闭环。
- 将 Story 1.2 已存在的 `SessionAuditFact`/`SessionAuditPort` 收敛到公共 `LocalAuditFact` 契约，保留登录、续期、登出、换号和宿主拒绝行为，避免并行建第二套审计机制。
- 为后续每个业务模块提供同一 schema、时间源、事务模式、迁移/权限模板和契约测试；各模块只在其首个受审计行为 Story 中创建自有表与端口，不在本 Story 空建九套未被使用的写路径。

## 实施前置与范围边界

1. 以当前工作树中的 Story 1.2 实现为真实基线；不得回退、覆盖或按 HEAD 重建这些用户改动。实施前记录状态，最终 File List 只声明本 Story 真实修改的文件。
2. 范围内：v1 `LocalAuditFact`/action catalog/可信时间 profile，模块自有 append-only 存储模式，关键命令与敏感读取事务门，identity-access 的真实表/端口接入，以及可被其他模块复用的迁移/权限 conformance fixture 与 PostgreSQL 原子性证据。
3. 范围外：集中 ledger、全局 sequence/previousHash、重放/积压安全门、审计检索 UI/API、归档、legal hold、DeletionReceipt 和备份过期。不得在本 Story 提前宣称 FR-8/NFR-15 全部完成。
4. 本 Story 不新增前端页面。未授权或审计依赖失败继续复用现有稳定错误包络和 `state-panel`；不新建审计上传/复核流程。
5. 不在业务事务内调用 NTP 或其他网络时间服务。运行时通过注入的可信时间端口获得 `Instant + source/profile + synchronization status`；时钟无法证明已同步时失败关闭。
6. 不将原始 IP、学生标识、token/cookie、证据正文、密钥、业务请求体或任意自由文本放入审计事实。操作人、对象和来源 IP 必须保存为可授权检索的版本化、域分离 search token（含 tokenization profile/key version），而不是不可查询的临时摘要；Story 1.5 可用相同受控端口从查询条件稳定派生 token，但不得获得密钥明文或跨域关联能力。

## Acceptance Criteria

### AC-1.3-HAPPY：上游受控验收

**Given** G-01 已批准且审计 schema 与可信时间源合同已冻结  
**When** 合规负责人执行本 Story 的主流程  
**Then** 业务提交或敏感返回与 `LocalAuditFact`、outbox 原子完成，审计失败时命令失败或内容不返回  
**And** 服务端授权、失败可见、审计、幂等和 `aggregateVersion` 冲突路径必须通过自动化或可复现验收。

### AC-1.3-CONTRACT：唯一的最小审计契约

**Given** 受控 v1 schema、action catalog、时间源 profile 与正反 fixture  
**When** 验证任一模块产生的本地审计事实  
**Then** 事实至少含 `auditId(UUIDv7),schemaVersion,producerModule,actorType,actorSearchToken,roleIds,authorizationContext,action,objectType,objectSearchToken,outcome,reasonCode,purpose/projectionScope,occurredAt,recordedAt,timeSourceProfile,sourceIpSearchToken,tokenizationProfileVersion/keyVersion,traceId,aggregateType,aggregateIdSearchToken,aggregateVersion,idempotencyKeyDigest,policy/profileVersions,retentionScheduleVersion`  
**And** 登录前/匿名拒绝可使用明确 `actorType=anonymous|service` 和空的不适用字段，不得伪造 role/grant/object  
**And** schema 拒绝未知字段、原始敏感值、无时区时间、未登记 action/result/reason 与不完整授权上下文；UUIDv7、时间 offset 等 `format` 由启用 format assertion 的验证器或独立语义 checker 强制，不把 JSON Schema 默认 annotation 当成校验通过；受控契约锁绑定所有文件 SHA-256，不允许静默漂移。

### AC-1.3-COMMAND-ATOMICITY：命令、幂等与审计同事务

**Given** 关键命令将聚合状态、幂等结果、必要的业务 outbox、`LocalAuditFact` 和 audit outbox 写入同一模块  
**When** 任一业务写入、审计写入、约束或 commit 失败  
**Then** 整个本地事务回滚，不返回成功、不更新聚合/幂等结果、不产生孤立审计或外发事实  
**And** 同一 `Idempotency-Key + request hash` 的普通传输重试只重放已提交结果，不重复原业务动作的 fact/audit-outbox；若需记录重试尝试，必须使用独立 attempt action/auditId 且不得伪造成第二次业务成功。安全语义的 replay/reuse（如 refresh-token reuse）、同键异请求和旧 `aggregateVersion` 必须返回稳定拒绝、状态不变，并各自以独立拒绝事实 + audit outbox 提交  
**And** 远程 IdP/工单/消息等外部副作用不得被假冒成数据库原子性；若外部状态已变而本地审计 commit 失败，必须失败关闭并进入显式重认证/恢复路径，不可返回假成功。

### AC-1.3-SENSITIVE-READ：返回敏感内容前持久提交

**Given** 读取已通过当前服务端授权且将返回敏感或对象级内容  
**When** 应用服务执行“重算授权 → 读取/投影 → append `LocalAuditFact` + audit outbox → commit”  
**Then** 只有事务成功 commit 后控制器/序列化器才能获得 DTO；审计、时钟、授权或 commit 失败时返回稳定失败包络，响应体、缓存、DOM 与无障碍树均不含对象内容  
**And** 以现有 `GET /api/v1/identity-sessions/current` 的当前会话投影作为首个真实接入路径，保持其授权重算、错误语义与响应 schema，并证明 audit commit 失败时不返回 `sessionPseudonym/sessionVersion/expiresAt`。

### AC-1.3-IDENTITY-MIGRATION：收敛 Story 1.2 的早期审计实现

**Given** identity-access 已有 `SessionAuditFact`、`SessionAuditPort`、`JdbcIdentityAccessStore.append(...)` 和 `ia_local_audit_fact`  
**When** 接入通用 v1 契约  
**Then** 采用向前兼容新迁移扩展现有表与数据，不改写/删除已有 `V000001__identity-access__session_boundary.sql`  
**And** 用唯一的 `LocalAuditFact` 类型和 identity-access 自有 application port/adapter 替代或显式适配旧 `SessionAuditFact/Port`；禁止长期保留两套并行 schema/表/写路径  
**And** 登录、refresh 成功/重放拒绝、logout、account-switch、版本/幂等拒绝和 host input rejection 保持原行为，每个已提交动作恰有一条符合新契约的事实和一条对应 audit outbox 记录  
**And** 不泄露 token/cookie/原始 IP/主体标识，不改变 Story 1.2 的 PKCE、续期旋转、安全 cookie、宿主契约、恢复壳和对外证据边界。

### AC-1.3-MODULE-OWNERSHIP：模块自有 append-only 模式

**Given** 九个受控模块的 schema/表前缀已登记在 `module-ownership.csv`  
**When** 建立本地审计存储  
**Then** identity-access 以新的全局六位迁移号扩展 `ia_local_audit_fact` 并建立独立 `ia_local_audit_outbox`，一次本地事务同时写事实和待归集事件；以可执行 conformance fixture 冻结“未来模块在自己的迁移、schema 和前缀中建表”的模式。本 Story 不为尚无接入行为的八个模块空建表，也不使用跨 schema 外键、共享业务表或可接受任意 schema/table 字符串的通用 SQL adapter  
**And** identity 在线应用身份只拥有必要的 `INSERT/SELECT`，不得 `UPDATE/DELETE/TRUNCATE` 事实表；`auditId/eventId` 唯一，投递/消费状态位于 outbox/delivery 边界，绝不通过改写原事实表达  
**And** v1 不定义跨模块或无缺口的 `producerSequence`：本地存储 surrogate 允许有 gap、不得充当完整性证明，1.4 按未归集集合与 `auditId/eventId` 幂等处理并由集中 writer 分配 `ledgerSequence`；若改变该决策，必须先补 ADR/契约版本  
**And** `LocalAuditFact` 是每个模块的本地技术事实，不将它添加为 `module-ownership.csv` 中只能有一个 owner 的领域 `factOwners`；集中 `AuditLedger` 仍唯一归 `audit-operations`。

### AC-1.3-TIME-AND-MINIMIZATION：可信时间与隐私负例

**Given** 注入的时间源 profile 声明 source id/version、UTC 口径、当前同步状态与 PP-1.0.0 的最大时钟偏差 100ms  
**When** 时间源未配置、状态不可信、返回倒退时间，或事实包含未批准字段/敏感值  
**Then** 关键命令失败、敏感内容不返回，并以不回显原值的稳定 code 显示依赖/契约失败  
**And** 测试明确证明原始学生/账号/IP/token/cookie/证据正文/密钥值不进入事实、日志、异常、fixture 或发布证据  
**And** 生产 adapter 从事务外采集并经受控绑定的时间同步状态读取 `source/profile,offsetMs,observedAt,freshUntil,evidenceRef`，仅在证据新鲜且 `abs(offsetMs) <= 100` 时供事务使用；静态配置、`Clock.systemUTC()` 或测试 fixed clock 均不得伪装生产通过  
**And** “倒退”只在同一节点/同一 source profile 的已接受观测窗口内检查，不承诺跨节点全局单调；`occurredAt` 表示受控业务发生时间，`recordedAt` 表示本地持久边界记录时间。重放依据 outbox 未归集集合和 `auditId/eventId`，不得从有 gap 的 surrogate 推导完整性，也不用未证明的 `Instant.now()` 或数据库 `clock_timestamp()` 绕过可信时间端口。

### AC-1.3-VERIFICATION：可复现的契约、事务与回归证据

**Given** 候选实现改变 Java、SQL migration、contract 和发布受控字节  
**When** 申请进入 `review`  
**Then** JSON Schema/action catalog/contract-lock 正反 fixture（含显式 format assertion/语义检查）、模块/迁移所有权、真实 PostgreSQL 18.4 事务回滚、只追加权限、幂等/冲突/并发/时钟/敏感读取失败测试全部通过  
**And** 完整 `scripts/verify.sh` 与两次 clean-root 可复现构建通过，新契约/依赖/迁移进入 source inventory、backend lock、SBOM、BuildManifest 和 ReleaseManifest 证据链  
**And** 不得用内存 fixture、H2/其他数据库或静态 SQL 搜索代替 PostgreSQL 18.4 的 commit/rollback/权限证据；无真实数据库证据时 Story 不得标记完成。

### 技术要求

- Java ID 继续复用现有 `UuidV7.generate(Instant)`；不得同时引入另一 UUID 库。事实时间必须来自注入端口，清除该路径中不可控的 `Instant.now()`。
- `LocalAuditFact`、audit outbox 与投递元数据分层：`recordedAt`/本地 surrogate 可由持久边界分配，但业务不得自行猜顺序；surrogate 允许有 gap 且不进入完整性承诺。集中 `ledgerSequence/previousHash` 不是本 Story 字段。
- action/outcome/reason 必须是版本化 catalog 中的稳定 machine code。不存自由文本错误、HTTP message、请求体或 stack trace。
- 角色、scope、grant 尚未实现时必须显式空/不适用并记录当时已用 policy/profile；不得预填 Story 1.6/1.7 未产生的授权事实。
- 业务表与审计表必须使用同一 local `DataSource`/transaction manager。跨模块同步调用或远程 API 不得加入该数据库事务。
- Spring 默认只对未检查异常回滚；若 application/adapter 捕获审计或数据库异常，必须重新抛出，或显式 `setRollbackOnly()` 后返回失败，禁止吞掉异常继续成功响应。
- actor、对象标识与 IP 使用规范化、keyed、版本化且域分离的 search token；禁止无密钥裸 hash、不可查询的随机摘要或跨用途复用同一 token。轮换时必须保留 RS-1.0.0 保留窗口内的授权查询能力，并由 1.5 的查询端口消费，而不是暴露 tokenization key。

### 架构合规

- 保持 `domain ← application ← adapters`。审计事实组装和用例编排在 application，JDBC/权限/SQL 在 outbound adapter，入站 HTTP 不直接写表。
- 跨模块不得导入另一模块的 application/domain/adapters。`audit-operations` 不获得本地业务表写权；1.4 只通过受控读取/事件边界归集。
- `shared` 只放 ID、时间、错误、trace、outbox 技术内核。不放角色、授权、业务 action policy 或任一模块 repository。
- 本地事实表归各模块；集中 AuditLedger/归档/检索投影仍由 audit-operations 唯一拥有。
- 迁移使用 `V<six-digit-global-sequence>__<owner-module>__<snake_case>.sql`，无跨 schema 外键，每个表前缀与 registry 一致，采用先扩展后收缩。

### 库与框架要求

- 保持受控基线 OpenJDK 25、Spring Boot 4.1.0/Spring Framework 受 BOM 管理版本、PostgreSQL 18.4、JSON Schema 2020-12、CloudEvents 1.0.2。不为本 Story 升级基线或手工覆盖 Spring 传递依赖版本。
- 继续使用已有 `TransactionTemplate`/`JdbcTemplate`；Spring 官方对 imperative flow 推荐 `TransactionTemplate`，回调完成后由 template 处理 commit/异常回滚。不混入 reactive transaction，不在新线程中假定事务上下文会自动传播。
- PostgreSQL 18 的 `current_timestamp/now()` 是事务开始时间，`clock_timestamp()` 是实时时钟。二者都不自动证明节点已按学校时源同步；审计业务时间仍必须由可测试端口/profile 提供，数据库只负责 UTC `timestamptz` 持久和顺序元数据。

# Epic 1 Context: 可信门户接入与可问责授权

<!-- Compiled from planning artifacts. Edit freely. Regenerate with compile-epic-context if planning docs change. -->

## Goal

建立可进入生产交付链的可信平台基础：复用学校统一门户与身份体系，以服务端角色、组织、责任关系和任务范围实施失败关闭授权；让关键命令与敏感读取产生不可分裂、可归集、可核验的审计证据；并为后续业务 Epic 提供稳定的模块边界、公共任务/消息适配器和可复现发布门禁。该 Epic 的价值不只是完成登录，而是确保身份、权限、字段投影、审计与发布证据在故障、重试、撤权和并发条件下仍保持一致。

## Stories

- Story 1.1a: 审计生产资产与供应链候选
- Story 1.1b: 建立模块化生产工程骨架
- Story 1.1c: 批准生产前端与性能 Profile ADR
- Story 1.1d: 固化 CI、供应链与质量门
- Story 1.2: 接入统一门户 SSO 与可恢复应用壳
- Story 1.3: 建立模块内原子审计事实
- Story 1.4: 归集不可变审计账本并检测篡改
- Story 1.5: 提供授权审计检索、归档与销毁证明
- Story 1.6a: 同步权威身份与组织
- Story 1.6b: 同步责任关系并执行每日对账
- Story 1.6c: 发布撤权、更正与失效传播事实
- Story 1.7: 提供角色化首页与组合对象授权
- Story 1.8: 执行字段级投影与任务期敏感授权
- Story 1.9: 建立公共任务、消息与回写适配器契约

## Requirements & Constraints

- 登录用户复用学校 SSO；会话失效后重新认证并返回原目标。身份、角色、组织和责任关系来自权威源，关系未知、冲突、过期或依赖不可用时必须拒绝访问。
- 服务端按角色、组织、责任学生、有效委托和协同任务逐请求计算对象范围。多角色采用 deny-overrides 与最严格字段投影，前端菜单或“角色切换”不能替代授权。
- 字段输出固定为明文、固定脱敏或完全隐藏三态；隐藏字段不得出现在 API、导出、DOM、缓存或无障碍树。策略、密钥或授权不可用时禁止降级为明文。
- 关键命令必须把业务状态、幂等结果、必要 outbox 与模块本地审计事实置于同一本地事务；敏感内容只能在读取审计提交后返回。审计失败不得返回假成功或泄露对象内容。
- 集中审计账本应按业务键幂等归集，以严格连续序号和前向哈希检测缺行、改行、重复与积压。原始记录仅追加；积压越过已发布安全阈值时，高风险读写失败关闭。
- 关键链路必须传播 traceId，技术日志、指标和告警不得包含学生明文、证据正文或密钥。审计记录必须能关联 HTTP、worker、outbox 和外部调用。
- 写请求使用 Idempotency-Key、请求摘要与 aggregateVersion；同键同请求重放已提交结果，同键异请求稳定冲突。重试、乱序、撤权和依赖故障必须有自动化负例。
- 每个候选制品必须通过受控构建、锁文件、测试、SBOM、签名、扫描与可复现验证；任何必需证据失败均阻断合并或发布。

## Technical Decisions

- 后端采用模块化单体与六边形边界，模块内部依赖方向为 `domain ← application ← adapters`。跨模块只能依赖公开 API/端口或版本化事件，禁止导入其他模块内部类型、直接访问其表或建立跨 schema 外键。
- `identity-access` 唯一拥有身份会话和本地审计事实；`audit-operations` 唯一拥有集中账本、完整性发现、归档状态与审计读模型。生产者保留自己的 outbox，中心不得扫描生产者 schema。
- 数据库使用模块独立 schema/前缀，迁移只归所属模块。业务状态与 outbox 同事务提交；跨模块投递至少一次，消费者以 eventId、业务键和 aggregateVersion 去重并拒绝旧版本覆盖。
- ID 使用 UUIDv7，数据库时间使用 UTC，接口时间使用带 offset 的 ISO 8601。错误码稳定且不泄露对象是否存在；并发冲突使用明确的版本或幂等错误。
- 同一后端制品按 `web-api` 与 `worker` 角色受控加载；正确性状态保存在 PostgreSQL，缓存只能加速。恢复后必须先完成序列、水位、对象与密钥对账，再开放写流量。
- 在线审计账本只能证明 tamper-evident，不等同于数据库 owner 无法重写整链；独立防篡改归档、保留和销毁证明由后续能力补齐，不得提前宣称完成。

## UX & Interaction Patterns

- 用户从统一门户或公共待办深链进入单一角色化首页；会话失效、无权限和对象失权时不泄露对象信息，并提供唯一明确的恢复动作。
- 系统运维表面只展示技术状态、策略版本、数据时间、积压和 traceId，默认不可见学生明细或证据正文。错误、空数据、筛选无结果、降级与陈旧状态必须分别表达。
- 审计检索原始记录只读；异步操作展示真实任务号、状态和失败原因，不能以 Toast 或客户端状态假装服务端已提交。
- 业务数据和授权投影只保留在当前进程内存；断网不展示缓存对象、不允许业务命令。重新联网后必须重新认证、重算授权并由用户显式重试。

## Cross-Story Dependencies

工程骨架、生产基线和 CI 供应链先于功能实现。SSO/会话为身份与授权入口；模块本地原子审计事实先于集中账本；集中账本先于授权检索、独立归档和销毁证明。权威身份、组织和责任关系同步完成后，角色化首页、组合对象授权和字段投影才能形成完整生产授权面。公共任务、消息与回写适配器为后续业务 Epic 提供集成契约，但不得反向拥有领域业务状态。
